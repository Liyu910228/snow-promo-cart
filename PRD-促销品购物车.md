> 版本：v1.2 | 日期：2026-04-24

## 一、需求概述

### 1.1 业务背景

总公司统一采购促销品，全国各地分公司业务员通过系统下单。每个促销品有：
- **最小起订量**：工厂最低生产数量
- **最小起送量**：单个仓储最低配送量

多个业务员可以跨仓储拼单，凑够最小起订量后触发生产。

### 1.2 核心价值

**管理员**：配置促销品信息、管理仓储、维护账号
**业务员**：浏览促销品 → 初选意向产品 → 加入购物车 → 查看拼单进度 → 合并下单

### 1.3 用户角色

| 角色 | 账号 | 密码 | 功能范围 |
|------|------|------|----------|
| 管理员 | admin | admin | 配置后台（促销品/最小起订量/最小起送量/仓储/账号管理） |
| 普通用户 | root | 12345 | 购物车 + 拼单功能 |

## 三、拼单规则（核心逻辑）

### 3.1 关键字段定义

| 字段 | 说明 | 示例 |
|------|------|------|
| 最小起订量 | 工厂最低生产数量（产品级） | 100台 |
| 最小起送量 | 单个仓储最低配送量（仓储级） | 30台 |

### 3.2 拼单逻辑

```
产品级（跨仓储汇总）：
  仓储A: 30个 + 仓储B: 30个 + 仓储C: 40个 = 100个 → 满足最小起订量

仓储级（单仓拼单）：
  深圳仓：已拼10个 / 最小起送量30个 → 还差20个可发货
```

**进度展示规则**：
- **仓储进度** = 已拼量 / 该仓最小起送量
- **全局进度** = 所有仓储已拼量之和 / 产品最小起订量

### 3.3 拼单状态

| 状态 | 条件 | 说明 |
|------|------|------|
| 🟡 待成团 | < 最小起订量 | 等待更多人加入 |
| 🟢 可发货 | ≥ 最小起订量 | 已满足最低要求，可继续加单 |

> ⚠️ "可发货"不是结束，满足100台后用户仍可继续加单

### 3.4 数量校验

| 场景 | 数量限制 | 说明 |
|------|----------|------|
| 用户下单数量 | 需手动输入，默认 **0**，输入后生效 | 用户必须输入数量 |
| 仓储发货 | ≥ 该仓**最小起送量** | 系统内部判断，用户无感知 |
| 跨仓生产 | 所有仓汇总 ≥ **最小起订量** | 系统内部判断，用户无感知 |

> ⚠️ 数量校验由系统内部处理，用户需手动输入数量（默认0），未输入时提示"请输入下单数量"
